Trav_Well Report

Project Description:

Name: Trav_Well

Problem: The problem we aimed to solve was that it is difficult to find information about where to visit at a given tourist destination.

Idea: The idea behind Trav_Well was to create a travelling website where users could get information about popular locations at a given destination. This information is mostly supplied by the users, adding a distinct social component to the site, and ensuring that locals can provide information about the best spots. In addition to being able to rate and share experiences at a particular location, users can also create a profile, add other users as friends, send messages to each other and even find people who wish to travel to similar places as them.

Load/Stress Testing

Security Measures

Protection against code injections: Codeigniter protects against sql and javascript injections, so the database is secure. Text fields are also limited to alpha-numeric characters so there should be no problems. This was done by Yuki.

Inputs have finite size: Users can only enter a limited number of characters for all input. Pictures uploaded to the site must be less than 300x300. This was done by Emmy.

Spam Protection: Although we do not protect against advertising/phishing in comments and messages, we do ensure that users can only post a limited number of comments and send a limited number of messages in a given time frame. This was done by Sean.

User interactions are finite: Users can only have a limited number of friends or messages in their inbox. Currently, for messages the oldest are deleted when a new one is sent if the user has too many . This could be inconvenient to the users. For friends, a user trying to add a new friend with a full friends list will get an error message. This was also done by Sean.

Protection against bots: In order to login or create a new user, users must fill out a captcha to ensure they are human. Although this is a bit annoying, it should protect against bots. This was done by Shen.

Spoofing: Users cannot add spaces to their usernames, and usernames must be unique. This is enforced by Codeigniter. This prevents users from spoofing other users for the most part. One possible workaround is to replace lower case 'l's with capital 'I's which may look the same in some fonts, but for the most part this should be sufficient. This was done by Shen.

Code Protection: A feature of Codeigniter is that unautherized users cannot access website files. This was done by Monica.

Login information is encrypted: Should prevent packet sniffing. This was also done via Codeigniter and was also done by Monica.

Project Limitations

Content must be overseen: We cannot trust users to add information about places. Any time a user adds a place to our database, we must verify that the information provided is legitimate. Ideally, we should be scraping the web to fill our database, or have some sort of process to automate information verification.

Matching based on interests: Right now, users are only shown similar users when they do not currently have any friends. Even then, it does not necessarily match based on intrests, only if both users wish to attend a similar place. Ideally, the matchmaking service would be more fleshed out and accessable at any time.

Spoofing protection: As stated above, there are still some potential issues with users spoofing other users. Ideally these would not exist.

Comments do not promote conversation: Currently, comments are simply listed one after another, as it is assumed users would mostly be posting reviews. In future, it would be nice if the design allowed users to comment on other user's reviews and have more in depth discussion.
